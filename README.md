## Overview

**Iris Deploy** is an end-to-end machine learning project that trains a classifier on the famous [Iris dataset](https://scikit-learn.org/stable/auto_examples/datasets/plot_iris_dataset.html) and serves it as a production-ready REST API inside a Docker container.

This project demonstrates a complete ML deployment workflow:

```
Train Model  →  Save Model  →  Build API  →  Dockerize  →  Deploy
```

It is ideal as a reference for anyone learning how to move a machine learning model from a Jupyter notebook into a real, deployable service.

---

##  Features

-  **Model training** on the Iris dataset using Scikit-learn
-  **Model persistence** — trained model saved to disk for serving
-  **REST API** to serve live predictions
-  **Fully Dockerized** — runs anywhere with a single command
-  **Clean dependency management** via `requirements.txt`
-  **Lightweight** — uses minimal, production-appropriate dependencies

##  About the Iris Dataset

The Iris dataset is one of the most well-known datasets in machine learning. It contains **150 samples** of iris flowers with **4 features** each:

| Feature | Description |
|---|---|
| `sepal_length` | Length of the sepal (cm) |
| `sepal_width` | Width of the sepal (cm) |
| `petal_length` | Length of the petal (cm) |
| `petal_width` | Width of the petal (cm) |

The model classifies each sample into one of **3 species**:
- *Iris Setosa*
- *Iris Versicolour*
- *Iris Virginica*

## Getting Started

### Prerequisites

- Python 3.8+
- Docker (for containerized deployment)

---

### Option A — Run Locally (without Docker)

#### 1. Clone the repository

```bash
git clone https://github.com/gouthamkumar8328/iris-deploy.git
cd iris-deploy
```

#### 2. Install dependencies

```bash
pip install -r requirements.txt
```

#### 3. Train the model

```bash
python train_model.py
```

This trains the classifier and saves the model file (`model.pkl`) into the `app/` directory.

#### 4. Start the API server

```bash
cd app
python main.py
```

The API will be available at `http://localhost:5000`.

---

### Option B — Run with Docker (Recommended)

#### 1. Clone the repository

```bash
git clone https://github.com/gouthamkumar8328/iris-deploy.git
cd iris-deploy
```

#### 2. Build the Docker image

```bash
docker build -t iris-deploy .
```

#### 3. Run the container

```bash
docker run -p 5000:5000 iris-deploy
```

The API will be available at `http://localhost:5000`.

##  API Usage

### Endpoint

```
POST /predict
```

### Request

Send a JSON body with the four Iris features:

```json
{
  "sepal_length": 5.1,
  "sepal_width": 3.5,
  "petal_length": 1.4,
  "petal_width": 0.2
}
```

### Response

```json
{
  "prediction": "Iris Setosa",
  "class_id": 0
}
```

### Example with curl

```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"sepal_length": 5.1, "sepal_width": 3.5, "petal_length": 1.4, "petal_width": 0.2}'
```

### Example with Python

```python
import requests

data = {
    "sepal_length": 5.1,
    "sepal_width": 3.5,
    "petal_length": 1.4,
    "petal_width": 0.2
}

response = requests.post("http://localhost:5000/predict", json=data)
print(response.json())
# Output: {"prediction": "Iris Setosa", "class_id": 0}
```

##  Docker Details

The `Dockerfile` defines the full build and run process:

```dockerfile
# Base image
FROM python:3.10-slim

# Set working directory
WORKDIR /app

# Install dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application files
COPY . .

# Train the model at build time
RUN python train_model.py

# Expose the API port
EXPOSE 5000

# Start the API server
CMD ["python", "app/main.py"]
```

### Useful Docker commands

```bash
# Build the image
docker build -t iris-deploy .

# Run in background (detached)
docker run -d -p 5000:5000 --name iris-api iris-deploy

# View running containers
docker ps

# View logs
docker logs iris-api

# Stop the container
docker stop iris-api

# Remove the container
docker rm iris-api
```

## 🚀 Deploying to the Cloud

Once containerized, this app can be deployed to any cloud provider:

### Render
1. Push your repo to GitHub
2. Go to [render.com](https://render.com) → New Web Service
3. Connect your repo and select **Docker** as the environment
4. Click **Deploy**

### Railway
```bash
railway up
```

### AWS / GCP / Azure
Push the Docker image to a container registry (ECR, GCR, or ACR) and deploy via the provider's container service.
